How a Hash Table Works

A hash function maps keys to bucket indices. Values are stored in the bucket. When two keys land on the same index, we have a collision.

Collision Resolution

Two strategies when multiple keys hash to the same bucket.

Chaining

Each bucket is a linked list. Colliding keys are appended.

"apple"
→
"cherry"
→
None

Pros: Simple, never full.
Cons: Extra memory, cache-unfriendly.

Open Addressing

Probe for the next empty slot in the array itself.

[3] apple
[3] ×
→
[4] cherry
[5] —

Pros: No pointers, cache-friendly.
Cons: Clustering, table fills up.

Common Interview Patterns

Five patterns that come up repeatedly in coding interviews involving hashing.

Frequency Map

Count occurrences with a dict. Use for anagram check, top-K frequent, etc.

Complement Lookup

Store seen values in a dict; for each new value check if target - val exists. Classic two-sum.

Sliding Window + Dict

Track counts or membership in a window using a dict or set as you slide.

Group by Key

Transform each item into a canonical key (e.g. sorted chars), group with defaultdict(list).

Set Operations

Union, intersection, difference to compare collections in O(n).
